# Project Overview

## What this project is

A link shortener application. Users authenticate, create short links, and are redirected via those short links.

## Tech Stack

| Layer           | Technology                                                 |
| --------------- | ---------------------------------------------------------- |
| Framework       | Next.js 16 (App Router)                                    |
| Language        | TypeScript 5 (strict mode)                                 |
| Runtime / React | React 19                                                   |
| Styling         | Tailwind CSS v4                                            |
| UI Components   | shadcn/ui (`base-nova` style) + Base UI (`@base-ui/react`) |
| Icons           | lucide-react                                               |
| Authentication  | Clerk v7 (`@clerk/nextjs`)                                 |
| Database        | Neon serverless PostgreSQL                                 |
| ORM             | Drizzle ORM                                                |
| Animations      | tw-animate-css                                             |

## Directory Structure

```
app/             # Next.js App Router pages and layouts
  globals.css    # Tailwind v4 CSS entry + design tokens (@theme)
  layout.tsx     # Root layout — ClerkProvider, fonts
  page.tsx       # Home page

components/
  ui/            # shadcn/ui components (generated — treat as owned code)

db/
  index.ts       # Drizzle client (Neon HTTP driver)
  schema.ts      # Drizzle table definitions

lib/
  utils.ts       # cn() helper (clsx + tailwind-merge)

proxy.ts         # Clerk middleware (Next.js middleware file — misnamed; should be middleware.ts if moved)
drizzle.config.ts # Drizzle Kit configuration
```

## Key conventions at a glance

- All imports use the `@/` path alias (maps to project root).
- UI primitives live in `components/ui/` and are generated by shadcn. Do not import from shadcn directly in feature code — always import from `@/components/ui/`.
- All database logic lives in `db/`. Schema changes require a Drizzle migration (`drizzle-kit generate` + `drizzle-kit migrate`).
- Authentication is handled entirely by Clerk — do not build custom auth logic.

## Non-negotiable rules

1. **App Router only.** There is no Pages Router. All routes live under `app/`.
2. **Server Components by default.** Only add `"use client"` when strictly necessary.
3. **Never bypass TypeScript.** No `any`, no `// @ts-ignore`.
4. **Database access is server-side only.** Never query the database from a Client Component.
5. **Auth is Clerk.** Do not write custom auth logic.
6. **Styling is Tailwind v4 + shadcn.** No `tailwind.config.js`. Use `cn()` to merge classes.
7. **No raw SQL.** Use the Drizzle query builder. Schema changes require a migration.
8. **Security.** Validate all inputs at Server Action and Route Handler boundaries. Scope all data queries to the authenticated user's ID.
